# FAQ / Частые вопросы

## Общие вопросы

### Что такое SellGPT?

SellGPT — это лендинг с личным кабинетом и админ-панелью для продажи доступа к ChatGPT Business. Проект включает форму заявок, систему управления лидами и интеграцию с Supabase.

### Какие технологии используются?

**Frontend:**
- React 18.2
- TypeScript 5.4
- Vite 5 (сборщик)
- CSS3 (кастомные стили)

**Backend:**
- Supabase (PostgreSQL + Auth)
- Google OAuth
- Row Level Security

**Шрифты:**
- Space Grotesk
- IBM Plex Sans / Mono

### Это официальный сайт OpenAI?

Нет. Это частное предложение на подключение ChatGPT Business. OpenAI не является стороной сделки.

## Установка и запуск

### Как запустить проект локально?

```bash
# 1. Клонировать репозиторий
git clone <url>
cd gpt

# 2. Установить зависимости
npm install

# 3. Создать .env файл с ключами Supabase

# 4. Запустить dev-сервер
npm run dev
```

### Какие требования к системе?

- Node.js 18 или выше
- npm 9 или выше
- Современный браузер (Chrome, Firefox, Safari, Edge)

### Где взять ключи Supabase?

1. Зарегистрируйтесь на [supabase.com](https://supabase.com)
2. Создайте новый проект
3. Перейдите в **Project Settings → API**
4. Скопируйте Project URL и anon key

## База данных

### Какая структура базы данных?

Проект использует две основные таблицы:

**profiles** — профили пользователей
- id, email, full_name, avatar_url, role

**leads** — заявки клиентов
- id, name, email, telegram, seats, company, goal, status, notes

### Как назначить администратора?

После первого входа пользователя выполните SQL:

```sql
update public.profiles 
set role = 'admin' 
where email = 'ваш-email@example.com';
```

### Как экспортировать заявки?

В админ-панели доступна кнопка **"Экспорт CSV"**. Экспортируются заявки с учетом текущих фильтров поиска.

### Безопасны ли данные?

Да. Используется:
- Row Level Security (RLS) — пользователи видят только свои данные
- Шифрование соединения (SSL/TLS)
- Защищенная аутентификация через Google OAuth

## Деплой

### Как задеплоить на Vercel?

1. Подключите репозиторий к Vercel
2. Настройки сборки:
   - Framework: Vite
   - Build: `npm run build`
   - Output: `dist`
3. Добавьте Environment Variables
4. Нажмите Deploy

Подробнее в файле [DEPLOY.md](./DEPLOY.md)

### Какой хостинг лучше использовать?

Рекомендуемые варианты:
1. **Vercel** — простой деплой, бесплатный SSL
2. **Netlify** — бесплатный план, формы
3. **Собственный сервер** — полный контроль

### Как подключить свой домен?

На Vercel:
1. Перейдите в Project Settings → Domains
2. Добавьте домен
3. Настройте DNS записи (A и CNAME)
4. Дождитесь проверки (обычно до 24 часов)

## Функционал

### Какие страницы есть в проекте?

- **index.html** — главная страница (base версия)
- **index-sales.html** — sales версия с агрессивным копирайтом
- **dashboard.html** — личный кабинет с админ-панелью
- **terms.html** — условия использования
- **privacy.html** — политика конфиденциальности

### Как работает админ-панель?

Админ-панель доступна после входа через Google пользователю с ролью `admin`. Включает:
- Статистику заявок
- Поиск и фильтрацию
- Экспорт в CSV
- Управление статусами и заметками
- Удаление заявок

### Как добавить новое поле в форму?

1. Добавьте поле в таблицу `leads`:
```sql
alter table public.leads add column phone text;
```

2. Добавьте input в форму (App.tsx)

3. Обновите обработчик отправки

### Поддерживается ли мобильная версия?

Да. Проект полностью адаптирован для мобильных устройств:
- Адаптивная верстка (mobile-first)
- Мобильное меню (бургер)
- Оптимизированные размеры шрифтов
- Touch-friendly интерфейс

## Устранение неполадок

### Ошибка: "Supabase не настроен"

Проверьте:
1. Файл `.env` существует
2. Переменные начинаются с `VITE_`
3. Значения корректны
4. Сервер перезапущен после изменений

### Ошибка: "Нет доступа к заявкам"

Пользователь должен иметь роль `admin` в таблице `profiles`.

### Ошибка: "Google OAuth не работает"

Проверьте:
1. OAuth настроен в Supabase
2. Redirect URI корректен
3. Google Cloud проект активен

### Медленная загрузка сайта

Рекомендации:
1. Включите gzip/brotli на сервере
2. Используйте CDN для статики
3. Оптимизируйте изображения
4. Включите кэширование

## Разработка

### Как добавить новую страницу?

1. Создайте HTML файл в корне
2. Добавьте в `vite.config.ts`:
```javascript
input: {
  // ...
  newpage: "newpage.html",
}
```

3. Добавьте обработку в `main.tsx`

### Как изменить цветовую схему?

Отредактируйте CSS переменные в `src/styles.css`:
```css
:root {
  --accent: #0b3b2e;      /* Основной цвет */
  --accent-2: #10c78a;    /* Вторичный */
  --bg: #f3f1eb;          /* Фон */
}
```

### Как добавить анимацию?

Используйте CSS классы:
- `.reveal` — появление при скролле
- `.pulse` — пульсирующий индикатор
- `data-reveal` — триггер для анимации

Или создайте свою анимацию:
```css
@keyframes myAnimation {
  from { opacity: 0; }
  to { opacity: 1; }
}
```
